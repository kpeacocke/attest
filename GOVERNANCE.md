# Governance

Attest is currently maintained by the project owner/maintainer.

## Decision making

- Small changes: pull request review and merge by a maintainer.
- Significant changes: open a **GitHub Discussion** (RFC-style) for community input.

## Workflow and tooling

Attest uses GitHub-native tools exclusively:

- **GitHub Issues** for bug tracking and feature requests
- **GitHub Discussions** for RFCs and community discussion
- **GitHub Projects** for milestone planning
- **GitHub Actions** for CI/CD automation
- **Snyk** for security scanning
- **SonarQube** for code quality analysis

We do not use JIRA, Trello, or other external project management platforms.
